**Apache 发布安全公告**  
  
  
  
近日，Apache发布了安全公告，修复了Tomcat中存在的安全漏洞，漏洞编号：CVE-2024-52318、CVE-2024-52317、CVE-2024-52316。  
  
Apache Tomcat是一种开源的Web应用服务器，用于提供基于Java的Web应用程序。它是由Apache软件基金会开发的，使用Java语言编写。  
  
**影响版本**  
  
**CVE-2024-52318**  
- Apache Tomcat 9.0.96  
  
- Apache Tomcat 10.1.31  
  
- Apache Tomcat 11.0.0  
  
**CVE-2024-52317**  
- 9.0.92 ≤ Apache Tomcat 9 ≤ 9.0.95  
  
- 10.1.27 ≤ Apache Tomcat 10 ≤ 10.1.30  
  
- 11.0.0-M23 ≤ Apache Tomcat 11 ≤ 11.0.0-M26  
  
**CVE-2024-52316**  
- 9.0.0-M1 ≤ Apache Tomcat 9 ≤ 9.9.95  
  
- 10.1.0-M1 ≤ Apache Tomcat 10 ≤ 10.1.30  
  
- 11.0.0-M1 ≤ Apache Tomcat 11 ≤ 11.0.0-M26  
  
  
  
**漏洞详情**  
  
**漏洞名称：**Apache Tomcat XSS漏洞  
  
**漏洞类型：**XSS  
  
**漏洞等级：**高危  
  
**CVE编号：**CVE-2024-52318  
  
**漏洞详情：**  
  
针对bug的修复导致pooled的jsp标签在使用后不被释放，这反过来可能导致某些标记的输出不按预期进行转义。这种未转义的输出可能导致XSS。  
  
  
**漏洞名称：**Apache Tomcat Request and/or response混淆漏洞  
  
**漏洞类型：**其它  
  
**漏洞等级：**高危  
  
**CVE编号：**CVE-2024-52317  
  
**漏洞详情：**  
  
HTTP/2请求使用的请求和响应的不正确回收可能会导致用户之间的请求和/或响应混淆。  
  
  
**漏洞名称：**Apache Tomcat 认证绕过漏洞  
  
**漏洞类型：**其它  
  
**漏洞等级：**低危  
  
**CVE编号：**CVE-2024-52316  
  
**漏洞详情：**  
  
如果将Tomcat配置为使用自定义Jakarta Authentication（以前称为JASPIC）ServerAuthContext组件，该组件可能会在身份验证过程中抛出异常，而不会显式设置HTTP状态以指示失败，则身份验证可能没有失败，从而允许用户绕过身份验证过程。没有已知的Jakarta Authentication组件以这种方式工作。  
  
**修复建议**  
  
**目前官方已发布相关安全版本，受影响用户可下载安装。**  
  
**参考链接**：  
  
https://tomcat.apache.org/index.html  
  
  
  
  
**Kafka****发布安全公告**  
  
  
  
近日，Kafka官方发布公告，公开了一个远程代码执行漏洞，漏洞编号：CVE-2024-31141。  
  
Kafka是由Apache软件基金会开发的一个开源流处理平台，由Scala和Java编写。该项目的目标是为处理实时数据提供一个统一、高吞吐、低延迟的平台。其持久化层本质上是一个“按照分布式事务日志架构的大规模发布/订阅消息队列”，这使它作为企业级基础设施来处理流式数据非常有价值。  
  
**影响版本**  
  
- 2.3.0 ≤ Apache Kafka ≤ 3.7.1  
  
  
  
**漏洞详情**  
  
**漏洞名称：**Apache Kafka 权限管理不当致信息泄露漏洞  
  
**漏洞类型：**其他  
  
**漏洞等级：**中危  
  
**CVE编号：**  
CVE-2024-31141  
  
**漏洞详情：**  
  
Apache Kafka客户端接受用于自定义行为的配置数据，并包含ConfigProvider插件以操作这些配置。Apache Kafka还提供FileConfigProvider、DirectoryConfigProvider和EnvVarConfigProvider实现，包括从磁盘或环境变量读取的能力。在Apache Kafka客户端配置可以由不受信任方指定的应用程序中，攻击者可以使用这些ConfigProviders读取磁盘和环境变量的任意内容。特别是，此漏洞可能会在Apache Kafka Connect中使用，以从REST API访问升级到文件系统/环境访问，这在某些环境中可能是不可取的，包括SaaS产品。  
  
**修复建议**  
  
**官方已发布安全版本，受影响用户可下载安装。**  
  
**参考链接**：  
  
https://github.com/apache/kafka/tags

**漏洞等级：**中危  
  
**CVE编号：**CVE-2024-52067  
  
**漏洞详情：**  
  
受影响版本会包括流同步过程中参数上下文值的可选调试日志记录，有权更改日志记录等级的授权管理员可以为框架流同步启用调试日志记录，从而导致应用程序将参数名称和值写入应用程序日志，参数上下文值可能包含敏感信息。使用默认Logback配置部署Apache NiFi不会记录参数上下文值。

https://nifi.apache.org/download.html
